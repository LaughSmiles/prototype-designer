## 2026年03月27日 14:30:00 - 修复侧边栏折叠后刷新页面展开按钮不显示的问题

### 目的
修复右侧页面库折叠后保存到 localStorage，刷新页面后展开按钮（chevron `<`）不显示，导致无法重新展开侧边栏的问题。

### 原因分析
`Storage.loadUIState()` 在初始化阶段通过 JS 给 sidebar 添加 `collapsed` class，但此时 `initSidebarResizer()` 中的 MutationObserver 尚未初始化，CSS 兄弟选择器也无法正确触发，导致 expander 按钮的 display 未被更新。

### 修复方案
在 `loadUIState()` 中，添加 `collapsed` class 后直接设置 expander 按钮的 `style.display`，与 `updateExpanderVisibility()` 的逻辑保持一致。

### 修改文件
- `js/storage.js` - `loadUIState()` 方法中增加 expander 按钮显示状态的手动设置

## 2026年03月27日 10:55:12 - 恢复悬浮预览功能（修复闪烁问题）

### 目的
恢复主题切换的悬浮预览功能，同时解决经典主题下预览时菜单闪烁、无法点击的问题。

### 根因
悬浮预览切换 `data-theme` 时，下拉菜单的 CSS 使用了 `var(--xxx)` 变量，变量值变化导致菜单背景/边框/阴影/尺寸发生变化，引发布局偏移 → 触发 `mouseleave` → 恢复主题 → 再次偏移 → 无限闪烁。

### 解决方案
将下拉菜单的所有样式改为**固定硬编码值**（白色背景、灰色边框、固定阴影），完全不依赖 CSS 变量。这样无论 `data-theme` 怎么切换，菜单本身的外观和尺寸都不会变，不会触发布局偏移。

### 修改文件
- `css/canvas-editor.css` - 下拉菜单样式改为固定值，移除经典主题覆盖
- `js/canvas-editor.js` - 恢复悬浮预览逻辑
