## 2026年03月27日 10:55:12 - 恢复悬浮预览功能（修复闪烁问题）

### 目的
恢复主题切换的悬浮预览功能，同时解决经典主题下预览时菜单闪烁、无法点击的问题。

### 根因
悬浮预览切换 `data-theme` 时，下拉菜单的 CSS 使用了 `var(--xxx)` 变量，变量值变化导致菜单背景/边框/阴影/尺寸发生变化，引发布局偏移 → 触发 `mouseleave` → 恢复主题 → 再次偏移 → 无限闪烁。

### 解决方案
将下拉菜单的所有样式改为**固定硬编码值**（白色背景、灰色边框、固定阴影），完全不依赖 CSS 变量。这样无论 `data-theme` 怎么切换，菜单本身的外观和尺寸都不会变，不会触发布局偏移。

### 修改文件
- `css/canvas-editor.css` - 下拉菜单样式改为固定值，移除经典主题覆盖
- `js/canvas-editor.js` - 恢复悬浮预览逻辑

## 2026年03月27日 10:50:34 - 修复经典主题下下拉菜单无法点击的问题

### 目的
经典主题下主题切换下拉菜单出现但无法点击。

### 根因分析
悬浮预览功能在切换 `data-theme` 时会改变整个页面 CSS，导致下拉菜单样式变化引发布局偏移，触发 `mouseleave` → 恢复主题 → 再次布局偏移 → 无限闪烁循环，使点击无法完成。此外经典主题的 `.status-bar` 设置了 `z-index: auto`，导致下拉菜单被后续 DOM 元素遮挡。

### 完成任务
- 移除悬浮预览功能（改为纯 hover 高亮 + 点击切换）
- 修复经典主题 `.status-bar` 的 `z-index` 从 `auto` 改为 `50`
- 清理不再使用的 `.previewing` CSS 类

### 修改文件
- `js/canvas-editor.js` - 移除 previewTheme 状态和 mouseenter/mouseleave 预览逻辑
- `css/canvas-editor.css` - 修复经典主题 status-bar z-index，移除 .previewing 样式
